Two management layers

Cloud API + SSH in one MCP server.

35

Cloud API Tools

Server power, metrics, snapshots, backups, firewalls, DNS zones and records, rescue mode, server rebuild and rescale — all through the Hetzner Cloud API. Works even when the server OS is unresponsive.

25

SSH Tools

Restart services, read logs, check disk and memory, manage Nginx configs, run read-only MySQL queries, inspect cron jobs, control supervisor processes, check UFW status — via direct SSH.

∞

Multi-Server

Configure 1 to N servers from a single MCP instance. Define names, aliases, and per-server SSH settings via environment variables. No code changes required.

Safety First

Confirm guards on all destructive operations. 29 blocked dangerous SSH command patterns. Read-only SQL enforcement. Tool annotations with readOnlyHint and destructiveHint on every tool.

Flexible Auth

Three modes: no auth (firewall-secured), API key, or API key + OAuth 2.1 with PKCE. Self-hosted, no mandatory accounts, no vendor lock-in.

Optional Layers

SSH tools and DNS tools register only when configured. Run API-only if you don’t need SSH. Add DNS management with a separate Hetzner DNS token. Graceful degradation by design.

Just ask

How it works in practice.

Connect the MCP to Claude.ai, Claude Desktop, or the Claude mobile app. Then ask questions in plain language. Claude uses the right tools automatically.

You
My website is returning 502 errors. Can you check what’s going on?
Diagnostics
Claude
I checked your production server. Nginx is running fine, but your app service myapp.service has crashed — it exited 4 minutes ago with an out-of-memory error.
The last 20 lines of the nginx error log confirm upstream connection failures. Your server is at 94% memory usage with only 180MB free.
Want me to restart the service and check if it recovers?
ssh_service_status ssh_logs_nginx_error ssh_memory_usage
You
Give me a quick health check across all my servers.
Monitoring
Claude
All 3 servers are running. Here’s the summary:
web — CPU 12%, RAM 2.1/4GB, disk 58% used, uptime 43 days
api — CPU 3%, RAM 1.4/4GB, disk 31% used, uptime 43 days
db — CPU 8%, RAM 5.2/8GB, disk 45% used, uptime 67 days
No immediate concerns. The db server has 12 pending security updates — want me to list them?
project_servers_list ssh_cpu_load ssh_memory_usage ssh_disk_usage ssh_uptime
You
I just deployed a new version. Restart nginx and check if the site loads.
Operations
Claude
Nginx config test passed. Reloaded successfully on the web server.
I checked the access log — requests are coming in and returning 200. The last 5 entries show normal response times. Your deployment looks good.
ssh_nginx_test ssh_nginx_reload ssh_logs_nginx_access
You
Show me which MySQL queries are running right now on the database server.
Database
Claude
There are 4 active connections on the db server. 3 are idle. One query has been running for 12 seconds:
SELECT * FROM orders WHERE created_at > ... on the shop_production database. It’s doing a full table scan — that table has 2.3M rows. Might need an index on created_at.
ssh_mysql_processlist ssh_mysql_query
You
Create a snapshot of the production server before I do a major update.
Backup
Claude
Snapshot created: web-pre-update-20260408. It’s currently in progress — the Hetzner API shows it at 30% completion. Should take another 2–3 minutes for a 40GB disk.
You now have 3 snapshots total. The oldest one is from March 15 — want me to delete it to save costs?
snapshot_create snapshot_list action_status

Get started in minutes.

Clone, configure, deploy. Works with any PHP 8.1+ server.

  • 1 Clone the repository
  • 2 Copy .env.example and add your Hetzner API token
  • 3 Run composer install
  • 4 Point Nginx to api.php
  • 5 Connect from Claude, Cursor, or VS Code
# clone
~ git clone https://github.com/wbf-solutions/hetzner-cloud-mcp.git
~ cd hetzner-cloud-mcp
 
# configure
~ cp .env.example .env
~ composer install
 
# connect from Claude Desktop
~ URL: https://your-server/api.php

About WBF.Solutions

Developer tools for cloud infrastructure.

WBF Solutions builds open-source tools for developers managing cloud servers. Based in Bucharest, Romania, with over 15 years of experience delivering web platforms and e-commerce solutions across Europe.

The Hetzner Cloud MCP was built to solve a specific operational gap: existing tools let you provision servers through the API, but none could connect to what’s actually running inside them. We manage production Hetzner infrastructure daily and needed SSH-level access — service control, log analysis, Nginx management, database queries — directly from AI-powered tools.

This is the result. Self-hosted, MIT-licensed, and used in production.
